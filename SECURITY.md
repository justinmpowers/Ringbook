# Security Policy

Ringbook is a self-hosted app that stores admin credentials, guest-uploaded
audio, and event metadata, so we take security reports seriously.

## Reporting a Vulnerability

Please **do not** open a public issue for security vulnerabilities.

Instead, use GitHub's private reporting feature:
[Report a vulnerability](https://github.com/justinmpowers/Ringbook/security/advisories/new)
(under the repository's **Security** tab → **Report a vulnerability**).

Include as much detail as you can:
- A description of the issue and its impact.
- Steps to reproduce, or a proof of concept.
- The version/commit you tested against.

This is a self-hosted, community-maintained project without a formal SLA,
but we'll do our best to acknowledge reports promptly and keep you updated
as we investigate and fix the issue.

## Supported Versions

Only the latest code on `main` (and the corresponding `latest` Docker image)
is supported. There are no maintained release branches.
